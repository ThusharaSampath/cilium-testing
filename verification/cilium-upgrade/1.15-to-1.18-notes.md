## Configmap diff vs OS 1.18 reference

### Critical (functional regressions)


1. **`cgroup-root`: AKS uses `/run/cilium/cgroupv2`; OS reference uses `/sys/fs/cgroup`.** This is environment-specific (OpenShift vs. AKS host layout). Don't carry the OS value into AKS.

### High (datapath behaviour change)

4. **`kube-proxy-replacement: "false"` → `"true"`.** Major datapath shift. Requires removing kube-proxy from the cluster (or accepting double-handling). Coordinate with cluster owner.

6. **`enable-node-port: "false"` → not set (defaults true with KPR).** Coupled to #5.

7. **`enable-host-port: "false"` → not set.** Coupled to #5.

8. **`enable-bandwidth-manager: "true"` → not set (defaults false).** AKS today uses bandwidth manager; OS does not.

9. **`enable-l2-neigh-discovery: "true"` → `"false"`.** 1.18 default flipped to `false`. If LB-IPAM L2 announcements (`CiliumL2AnnouncementPolicy`) are in use, they break. None observed today.

10. **`enable-endpoint-routes: "true"` (new on OS).** Per-endpoint routes vs. `cilium_host`. Different pod-to-pod path. Re-validate `cross-node-test.sh` after rollout.

### Medium (new defaults, semantic changes)

11. **`enable-non-default-deny-policies: "true"` (new in 1.16).** Empty non-nil selectors (`fromEndpoints: []`, etc.) now mean deny instead of allow. Audit result: 0 hits across all 306 CNPs and 7 CCNPs. Not a problem.

12. **`tofqdns-endpoint-max-ip-per-hostname: 50 → 1000`.** Bigger memory footprint per agent. Acceptable.

13. **`identity-management-mode: agent` (new in 1.18).** Default; fine.

14. **Hubble metrics surface on OS 1.18:** `hubble-metrics-server: :9965`, `hubble-metrics: httpV2:destinationContext=pod;sourceContext=pod;http_requests_total=status;http_request_duration_seconds=status`, `hubble-network-policy-correlation-enabled: "true"`. Service `hubble-metrics` (port 9965, clusterIP None) and `ServiceMonitor` `hubble` in `choreo-observability` are wired up. `hubble-observability-test.sh` passes against this. File-based event export (`hubble-export-file-*`, `hubble-export-allowlist`/`denylist`, `hubble-event-buffer-capacity`, `enable-hubble-open-metrics`) used on AKS 1.15 does **not** exist on OS — no test depends on it.

### Low / informational

15. Removed-in-1.16 keys, absent from the OS 1.18 config (good): `enable-remote-node-identity`, `sidecar-istio-proxy-image`, `arping-refresh-period` (also removed in 1.18).

16. Deprecated-in-1.18 keys still set explicitly on OS 1.18 (will need removal at next bump): `enable-svc-source-range-check`, `enable-health-check-nodeport`, `enable-health-check-loadbalancer-ip`, `enable-internal-traffic-policy`. Harmless on 1.18.

17. AKS 1.15 configmap carries `kapp.k14s.io/*` and `checkov.io/skip*` annotations; OS does not. If kapp manages the AKS configmap, the apply path for the upgraded version must preserve kapp ownership semantics.

18. `policy-default-local-cluster: "false"` — 1.19 will flip this default. Run `cilium clustermesh inspect-policy-default-local-cluster --all-namespaces` ahead of the next upgrade. Not blocking now.
